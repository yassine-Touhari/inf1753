## Règles de contribution

Branches :
- main : version stable
- dev : version en développement
- feature/<nom> : travail en cours

Commits :
- Format recommandé : <type>: <description>
- Exemples : add:, update:, fix:, docs:

Pull Requests :
- Une revue obligatoire avant d’intégrer dans main.
- Une issue doit être référencée dans chaque PR.
